Our Dual token system

What can we do with it

Dual Token system

All our Private tokens work as stable tokens and are only usable inside that specific ecosystem. Each project signing up will have their own private tokens, their own vaults and decide on their own how to distribute value through their system. These two tokens both have their own vault with their own underlying assets. Tokens only get minted when value is inside the vault. Each Eco system also receives the tools for NFT membership controls.

These vaults hold the Security tokens of the Entity (Equity) and our tokens are a utility layer for that equity locked inside the vault.

wForce Token

The wForce token (wForce) - A token that you earn for contributing your time and efforts towards the DG, any task or responsibility will have a reward and for each task you complete you can claim your reward. Your work is validated by others and together they sign off on completion. The value behind these tokens will need to come from revenue streams only or the initial burn at the staking with the sKing token. wForce tokens are minted each month when receiving their revenue share at a fixed ratio per token that can only change by vote of the Council with a max inflation or deflation of 5% per month announced 1 month prior to the change. 

sKing Token

The sKing token(sKing) is used when wForce holders want to gain passive income through staking to secure the position of the dGuild, they will sell x amount of Security Tokens towards the vault of the sKing token, they can select how much of this value is distributed towards the other vaults (this process is called “The Burn”, that value is not claimable any more by sKing holders). The remaining value gets locked in the sKing vault and is used in staking, this gives a passive income towards the dGuild but stays claimable for the sKing holder at any time (with a 7-day unlock period).


Funding rounds

We plan to have 2 funding rounds.
The first round is going through a NFT sale, we can't make any promises on utility but we have some ideas we present later.

We try to raise 3.000 SOL and this would go entirely towards NFT funding vault. 
These funds will be used to create the legal framework and allow us to start our tokenomics as presented. 

From there we build MVP and allow other Guilds to start their own Ecosystem. 

The second round will be our sKing token and this vault will receive 20% of the equity shares, we want to raise 10.000 SOL in this round. (''the Burn'' applied). This money would be used to build MVP and grow the platform, build towards support for other blockchains beside Solana. To participate in this round you need to be holding a NFT from the funding round. 


 '' the Burn ''

When the sKing token is initially converted, only 70% of that value gets stored inside the vault. The remaining 30% goes towards the other vaults for bootstrapping the Ecosystem.  Each ecosystem gets to decide on their own how these variables are set.
The remaining 70% will go into the sKing vault for staking and used for a passive income. You can claim back your initial 70% + revenue shared by converting your sKing back to the Vault.

Distributions from Ecosystem

The dGuild vault collects all value and deducts the cost of operation. The is then spread amongst all treasury. Our own variables for this are, this is based on the STO tokens, both vaults give half their value towards the dGuild vault:
- 10% sKing vault
- 40% wForce vault
- 50% dGuild vault
But each eco system is able to set their own values. All variables set are saved and stored inside a NFT locked inside the dGuild vault.
